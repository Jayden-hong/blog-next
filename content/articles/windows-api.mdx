## 核心观点提炼
Windows API的ReadDirectoryChangesW监控功能存在隐患：文件被删除后，无法获取相关属性。微软工程师揭示了维护缓存或使用ReadDirectoryChangesExW来填补这一信息盲区的必要性。

## 关键细节展开
在使用Windows API的ReadDirectoryChangesW函数监控目录变化时，我们可能会遇到一个棘手的问题：当文件被删除并触发通知，我们却无法获取到这些文件的属性信息。这个问题背后的原因，是Windows的文件系统通知机制与文件属性查询之间的一个设计缺口。

微软的资深工程师透露，解决这个问题的关键在于维护一个文件属性的缓存。每当文件发生变化时，这个缓存就会被更新，确保即使文件被删除，我们也能通过缓存来访问其属性信息。此外，他还提到了另一个解决方案，即使用ReadDirectoryChangesExW函数，这个函数提供了额外的控制，允许我们更精细地管理文件属性的查询和更新。

## 影响分析/延伸思考
这一发现对于依赖文件系统监控的应用程序尤为重要，尤其是在需要处理大量文件和频繁变更的环境中。通过维护文件属性缓存或使用ReadDirectoryChangesExW，开发者可以确保即使在文件被删除的情况下，也能准确地追踪和记录文件的属性变化，从而提高应用程序的稳定性和可靠性。

对于整个开发社区而言，这不仅是一个技术问题的解决方案，也是一个提醒：在设计文件系统监控方案时，需要考虑到Windows API的这些潜在限制，并寻找合适的方法来规避或解决它们。这将有助于提升整个行业的技术实践和产品质量。
